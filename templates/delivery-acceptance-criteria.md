# Delivery Acceptance Criteria

## 1. 代码类交付

- 可以运行
- 有清晰的运行命令
- 核心路径已测试
- 错误处理覆盖外部输入和异常状态
- 没有硬编码敏感信息
- 不引入无必要依赖
- 不破坏既有功能
- 有必要的回归验证说明

## 2. 前端类交付

- 页面符合需求和设计目标
- loading / error / empty 状态明确
- 核心交互可用
- 响应式或目标设备已验证
- 浏览器控制台无明显错误
- 涉及表单时有输入校验和错误反馈

## 3. 后端/API 类交付

- 接口请求和响应结构清楚
- 鉴权和权限边界明确
- 业务异常有可理解错误码或错误信息
- 数据写入具备一致性考虑
- 核心接口有测试或可复现验证命令

## 4. 数据库类交付

- 表结构和索引设计说明清楚
- 迁移脚本可执行
- 有回滚或恢复方案
- 兼容已有数据
- 高风险操作已标注

## 5. 测试类交付

- 覆盖核心成功路径
- 覆盖关键异常路径
- 说明测试环境和测试数据
- 输出测试结果
- Bug 修复类任务包含复现和回归方式

## 6. 部署类交付

- 构建命令明确
- 环境变量清单明确
- 部署步骤可复现
- 健康检查明确
- 日志位置明确
- 回滚方案明确

## 7. 文档类交付

- 背景清楚
- 输入输出明确
- 步骤可复现
- 风险和限制已说明
- 目标读者能直接使用

## 8. 安全类交付

- 授权范围明确
- 测试边界明确
- 复现步骤完整
- 风险等级清楚
- 修复建议可执行
- 证据充分但不泄露敏感数据
- 不包含破坏性或未授权利用内容
