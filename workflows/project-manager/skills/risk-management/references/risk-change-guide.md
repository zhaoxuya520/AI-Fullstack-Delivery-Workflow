# 风险和变更管理指南

## 1. 风险登记

每个风险至少包含：

```text
风险描述
类型
概率
影响
触发条件
缓解方案
负责人
状态
```

风险等级建议：

```text
高概率 + 高影响 = 严重
高概率 + 中影响 = 高
中概率 + 高影响 = 高
其他按中/低处理
```

---

## 2. 风险类型

```text
需求风险：范围不清、验收不清、优先级不清。
技术风险：方案未知、依赖复杂、性能不确定。
安全风险：权限、敏感数据、攻击面、合规。
交付风险：时间不足、资源冲突、返工高。
外部风险：第三方 API、审批、账号、证书、环境。
```

---

## 3. 阻塞升级

```text
L1 自动解决：工具短暂不可用、小范围缺失，5 分钟内处理。
L2 协调解决：跨工作流产物不一致、依赖延迟，15 分钟内协调。
L3 人工介入：需求不清、授权缺失、外部依赖不可控，通知用户。
```

---

## 4. 变更控制

变更处理流程：

```text
描述变更
→ 判断是否必须现在做
→ 评估影响范围
→ 评估工期和风险
→ 决策接受/拒绝/后置
→ 更新计划和通知下游
```

---

## 5. 变更决策规则

```text
不影响关键路径且价值高：可接受。
影响关键路径但必须上线：接受并调整里程碑。
价值不高但成本高：后置。
需求不清：退回产品经理工作流。
涉及安全/合规：必须先评审再决定。
```

---

## 6. 项目复盘中的风险问题

复盘时检查：

```text
哪些风险提前识别了？
哪些风险没有识别？
哪些风险处理有效？
哪些门禁应该提前？
哪些风险模式要写入 pitfalls.md？
```
