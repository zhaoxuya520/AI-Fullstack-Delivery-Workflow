# 发布计划模板

## 1. 发布信息

```text
版本：
日期：
负责人：
审批人：
```

## 2. 发布策略

```text
策略：滚动 / 蓝绿 / 金丝雀 / Feature Flag
灰度比例：5% → 50% → 100%
观察时间：每阶段 X 分钟
```

## 3. 前置条件

```text
□ CI 全绿
□ staging 验证通过
□ 数据库迁移已执行
□ 环境变量已配置
□ 监控告警就绪
□ 回滚方案确认
□ 审批通过
```

## 4. 回滚方案

```text
触发条件：错误率 > X% / P99 > Xms
回滚方式：kubectl rollout undo / 切回旧版本
回滚时间：< 5 min
数据回滚：[是否需要 / 方案]
```

## 5. 监控指标

```text
□ 错误率（vs 基线）
□ P99 响应时间
□ CPU / Memory
□ 业务指标（订单数 / 注册数）
□ 日志异常
```

## 6. 发布步骤

| 步骤 | 操作 | 验证 | 回滚 |
|---|---|---|---|
| 1 | 部署 staging |  |  |
| 2 | staging 验证 |  |  |
| 3 | 部署 prod 5% |  |  |
| 4 | 监控 5 min |  |  |
| 5 | 扩到 50% |  |  |
| 6 | 监控 10 min |  |  |
| 7 | 扩到 100% |  |  |
| 8 | 观察 1 小时 |  |  |

## 7. 发布后

```text
□ 监控正常
□ 业务指标正常
□ 通知团队
□ 更新文档
□ 清理旧版本
```
