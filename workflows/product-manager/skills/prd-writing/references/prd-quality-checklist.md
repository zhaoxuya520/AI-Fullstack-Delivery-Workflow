# PRD 质量检查清单

## 1. 背景和目标

- [ ] 是否说明为什么要做？
- [ ] 是否说明当前问题？
- [ ] 是否说明不做的影响？
- [ ] 是否有业务目标？
- [ ] 是否有用户目标？
- [ ] 是否区分事实、假设和待验证问题？

## 2. 用户和场景

- [ ] 目标用户是否明确？
- [ ] 使用场景是否明确？
- [ ] 触发条件是否明确？
- [ ] 用户当前替代方案是否明确？
- [ ] 用户需求是否从用户视角表达？
- [ ] 是否有用户旅程或关键流程？

## 3. 产品发现

- [ ] 是否列出关键机会点？
- [ ] 是否列出候选方案？
- [ ] 是否标注关键假设？
- [ ] 是否说明验证方式或证据来源？
- [ ] 是否检查价值、可用性、可行性、商业可行性风险？

## 4. 范围

- [ ] 本版本做什么是否明确？
- [ ] 本版本不做什么是否明确？
- [ ] 后续版本候选是否明确？
- [ ] MVP 是否能形成最小闭环？
- [ ] 是否有路线图或版本边界？

## 5. 功能和规则

- [ ] 每个功能是否有用户价值？
- [ ] 用户故事是否满足 INVEST 基本检查？
- [ ] 是否有业务规则？
- [ ] 是否有角色权限？
- [ ] 是否有状态流转？
- [ ] 是否有异常场景？

## 6. 验收

- [ ] 是否有 Given/When/Then？
- [ ] 是否覆盖成功路径？
- [ ] 是否覆盖失败路径？
- [ ] 是否覆盖边界条件？
- [ ] QA 是否能直接转成测试用例？

## 7. 指标

- [ ] 是否定义目标、信号、指标？
- [ ] 是否说明数据来源？
- [ ] 是否说明观察周期？
- [ ] 是否定义成功标准？
- [ ] 是否有防护指标？
- [ ] 是否有埋点建议？

## 8. 下游交接

- [ ] UI/UX 是否能开始设计？
- [ ] API 是否能开始定义接口？
- [ ] 数据库是否能识别实体？
- [ ] 前后端是否能拆任务？
- [ ] QA 是否能设计测试？
- [ ] 安全是否能识别风险？
- [ ] 数据分析是否能定义埋点和复盘？

## 9. 风险

- [ ] 是否有业务风险？
- [ ] 是否有技术风险？
- [ ] 是否有体验风险？
- [ ] 是否有安全风险？
- [ ] 是否有交付风险？
- [ ] 是否有未决问题清单？
