# 数据分析工作流常见坑

> 当前文件是未开写工作流的标准坑位模板。正式完善本章节时，应替换为岗位专属高频坑，并按“表现 / 风险 / 避免方式”展开。

## 1. 输入不足就开始执行

- 表现：没有确认目标、上游交付物和验收标准就开始产出。
- 风险：交付物无法被下游接手，返工成本高。
- 避免方式：按 `WORKFLOW.md` 的输入不足补问规则先补齐关键信息。

## 2. 只交付内容，不交付验证

- 表现：只输出方案或代码，没有验证结果。
- 风险：下游无法判断是否可用。
- 避免方式：每次交付必须附验证结果；无法验证时说明原因和建议验证方式。

## 3. 忽略跨工作流边界

- 表现：把 API、数据库、测试、部署或安全问题混在当前工作流内直接假设。
- 风险：职责不清，关键风险漏掉。
- 避免方式：按 `routing.md` 转出规则交给对应工作流或标注待确认。

## 4. 模板长期停留在占位状态

- 表现：`skills/_template` 没有被拆成真实子 skills。
- 风险：工作流无法承载复杂任务。
- 避免方式：正式开始本章节时先定义 skills 路由矩阵，再下沉模板和参考资料。
