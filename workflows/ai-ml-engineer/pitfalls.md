# AI/算法工程师常见坑

## 1. 没有基线就优化
- 风险：不知道改进了多少。
- 避免：先跑简单基线（规则 / 随机）。

## 2. 数据泄露
- 风险：测试集信息泄露到训练。
- 避免：严格拆分 + 时间序列注意。

## 3. 过拟合
- 风险：训练好测试差。
- 避免：正则化 + 交叉验证。

## 4. 不做离线评估就上线
- 风险：线上效果未知。
- 避免：离线 + A/B 两阶段。

## 5. Prompt 不版本化
- 风险：效果退化无法追溯。
- 避免：Prompt 进版本控制。

## 6. RAG 不评估召回率
- 风险：检索不到相关内容。
- 避免：用标注数据评估 Recall@K。

## 7. 向量库不更新
- 风险：知识过期。
- 避免：增量更新 + TTL。

## 8. GPU 浪费
- 风险：成本爆炸。
- 避免：按需调度 + 量化。

## 9. 不监控线上效果
- 风险：模型退化不知道。
- 避免：日志 + 指标 + 告警。

## 10. 幻觉不处理
- 风险：LLM 输出错误信息。
- 避免：RAG + 置信度 + 人工审核。
