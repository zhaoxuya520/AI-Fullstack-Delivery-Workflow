# 技术文档常见坑

## 1. 写完不更新
- 风险：过期文档 = 误导。
- 避免：文档跟代码一起发版。

## 2. 没有示例
- 风险：读者看不懂。
- 避免：每个概念至少 1 个示例。

## 3. 太长没导航
- 风险：找不到需要的内容。
- 避免：目录 + 锚点 + 搜索。

## 4. 术语不一致
- 风险：混淆读者。
- 避免：术语表 + 全文统一。

## 5. 敏感信息泄露
- 风险：密钥 / 内部 URL 进公开文档。
- 避免：审查 + 占位符。

## 6. 命令不可复制
- 风险：路径 / 环境不一致。
- 避免：用通用路径 + 说明前置条件。

## 7. 缺少前置条件
- 风险：读者跟着做卡住。
- 避免：列出依赖 / 版本 / 权限。

## 8. 不区分受众
- 风险：开发文档给 PM 看不懂。
- 避免：明确读者 + 分层。
