# API 设计方法指南

## 1. 契约优先

```text
先写契约，再写实现。
前端、后端、QA、安全、文档都以契约为共同边界。
OpenAPI 描述机器可读结构，Markdown 描述设计意图和交接背景。
```

## 2. 从用户任务到 API

```text
用户任务
→ 页面/调用场景
→ 业务资源
→ 资源操作
→ 端点
→ 请求响应
→ 错误和权限
→ Mock 和验收
```

## 3. API 设计层次

```text
资源模型：业务对象和关系
端点清单：对外能力边界
Schema：请求响应结构
错误模型：失败和恢复
权限模型：谁能做什么
可靠性：幂等、重试、限流、并发
交接：Mock、示例、验收
```

## 4. API-first 检查

```text
前端能否只靠 Mock 开发？
后端能否只靠契约实现？
QA 能否根据契约设计测试？
安全能否根据契约评估越权和敏感数据？
文档能否根据契约生成调用说明？
```
