## 4. 联调步骤

```text
1. 前端接入 Mock 地址。
2. 使用示例 Token 或测试账号。
3. 验证主路径。
4. 验证失败路径。
5. 切换真实后端地址。
6. 对比 Mock 与真实响应差异。
```

---

## 5. 验收标准

```text
OpenAPI 与 Mock 字段一致。
Mock 覆盖主路径和失败路径。
前端可独立完成页面主流程。
QA 可根据 Mock 设计接口测试。
```
