# Mock 和交接指南

## 1. Mock 原则

```text
Mock 来自 OpenAPI Schema。
Mock 覆盖成功、空、错误、权限、冲突和限流。
Mock 数据不包含真实敏感信息。
Mock 与真实后端差异必须记录。
```

## 2. 联调交接

交给前端：

```text
OpenAPI 文件
Mock 地址
认证方式
示例请求响应
错误处理建议
联调步骤
```

交给后端：

```text
资源模型
端点清单
Schema
权限规则
幂等重试
Webhook/异步
```

交给 QA：

```text
成功路径
失败路径
权限场景
边界数据
并发和重试场景
```

交给安全：

```text
认证鉴权
租户隔离
敏感字段
Webhook 签名
限流和滥用风险
```

交给技术文档：

```text
对外说明
示例
错误码
版本变更
迁移指南
```

## 3. 验收

```text
前端能用 Mock 完成主路径。
后端实现能通过契约检查。
QA 能根据契约写接口测试。
安全能根据契约评审权限和敏感数据。
文档能根据契约生成调用说明。
```
