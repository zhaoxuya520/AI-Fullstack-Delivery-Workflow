# 幂等和重试指南

## 1. 需要幂等的场景

```text
创建订单
提交审批
支付
扣减余额或库存
发送通知
导入任务
任何可能因重试重复执行的写操作
```

## 2. 幂等键

```text
Header：Idempotency-Key
来源：客户端生成 UUID 或业务唯一单号
作用域：用户 + 端点 + key
保存窗口：按业务风险定义
重复请求：返回首次处理结果或处理中状态
```

## 3. 重试规则

```text
安全重试：GET、幂等 PUT、带幂等键的 POST
谨慎重试：支付、扣减、审批
不可重试：参数错误、权限错误、业务规则失败
```

## 4. 限流

```text
状态码：429
Header：Retry-After
响应体：错误码、恢复建议、requestId
```

## 5. 并发冲突

```text
状态码：409
检测方式：version、ETag、updatedAt、业务锁
恢复方式：刷新后重试、合并后提交、放弃修改
```

## 6. 超时处理

```text
提交超时后不要盲目重复提交。
提供查询接口确认最终结果。
异步任务返回 taskId 或 operationId。
```
