# 错误和认证鉴权指南

## 1. HTTP 状态码

```text
400 参数格式或校验错误
401 未认证或认证无效
403 已认证但无权限
404 资源不存在或不可见
409 资源状态冲突或并发冲突
422 业务语义校验失败
429 请求过于频繁
500 服务端未知错误
```

## 2. 错误响应结构

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "参数校验失败",
    "details": [],
    "requestId": "req_xxx"
  }
}
```

## 3. 调用方恢复

错误设计必须回答：

```text
用户能否修正？
前端应该展示什么？
是否可以重试？
是否需要重新登录？
是否需要刷新资源状态？
是否要联系管理员？
```

## 4. 认证和鉴权区别

```text
认证：调用者是谁。
鉴权：调用者能否执行操作。
租户隔离：调用者是否属于同一组织或空间。
资源归属：调用者是否拥有或被授权访问资源。
字段权限：调用者能否查看或修改特定字段。
```

## 5. 安全注意

```text
不要在错误中返回敏感内部信息。
不要区分过细导致可枚举用户或资源。
权限错误和资源不存在的返回策略要按安全要求选择。
所有安全相关失败应有 requestId 便于审计。
```
