## 按任务类型路由

| 用户任务 | 是否进入本工作流 | 说明 |
|---|---|---|
| 设计接口、端点、请求响应 | 是 | API 设计主责 |
| 写 OpenAPI / Swagger | 是 | 输出机器可读契约 |
| 定义错误码和 HTTP 状态码 | 是 | 与 QA、前端、后端协作 |
| 设计认证、鉴权、租户隔离 | 是 | 同时转交安全工作流评审 |
| 设计分页、筛选、排序 | 是 | 需要对接 UI 表格和后端查询能力 |
| 设计 Webhook 或异步任务 | 是 | 需要定义事件、签名、重试、查询状态 |
| 写后端代码实现 | 否 | 转后端工程师工作流 |
| 设计数据库表结构 | 否 | 转数据库工程师工作流 |
| 设计页面和交互 | 否 | 转 UI/UX 设计工作流 |
| 写测试用例或执行测试 | 否 | 转 QA 工作流 |

## 进入前补问规则

输入不足时先问：

```text
1. 这个 API 支持哪个业务能力或页面？
2. 调用方是谁：前端、移动端、第三方、内部服务还是后台任务？
3. 核心资源是什么？资源之间有什么关系？
4. 哪些角色可以访问？是否有租户、组织或资源归属限制？
5. 需要分页、筛选、排序、搜索、导出吗？
6. 是否涉及创建、提交、支付、扣减、审批等需要幂等的操作？
7. 是否涉及 Webhook、回调、异步任务或轮询？
```

---

## 转出规则

```text
需求范围不清 → 产品经理工作流
页面字段和状态不清 → UI/UX 设计工作流
实体关系和存储约束不清 → 数据库工程师工作流
需要实现接口 → 后端工程师工作流
需要调用接口 → 前端工程师工作流
需要接口测试 → QA 工作流
涉及认证安全、敏感数据、越权风险 → 安全工程师工作流
需要发布、网关、监控 → DevOps / SRE 工作流
```
