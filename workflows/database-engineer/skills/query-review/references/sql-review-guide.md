# SQL 审查指南

## 审查维度

1. 查询是否匹配业务场景。
2. 过滤条件是否可利用索引。
3. 排序、分页、聚合是否会产生大范围扫描。
4. join 关系是否基于合理基数和索引。
5. ORM 是否产生 N+1 或额外查询。
6. 事务范围是否过大，是否存在锁等待。

## 证据优先级

```text
真实慢日志 / 监控
→ EXPLAIN / EXPLAIN ANALYZE
→ 数据量和参数分布
→ 本地推测
```

## 常见优化方向

- 查询重写：减少无效 join、减少返回字段、拆分冷热路径。
- 索引调整：复合索引、覆盖索引、唯一索引、局部索引。
- 分页调整：深分页改 cursor 或 seek pagination。
- 架构调整：把报表聚合从在线链路拆出。
