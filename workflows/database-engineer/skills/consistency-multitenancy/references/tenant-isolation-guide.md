# 多租户与一致性指南

## 多租户隔离层次

| 层次 | 说明 | 适用场景 |
|------|------|----------|
| 共享库共享表 | 通过 tenant_id 隔离 | 中小型 SaaS |
| 共享库独立 schema | schema 级隔离 | 租户规模较大 |
| 独立库 | 每租户独立数据库 | 强隔离或企业客户 |

## 共享表关键规则

- 租户数据表必须明确租户键。
- 查询默认带租户过滤。
- 业务唯一约束通常应包含租户键。
- 后台跨租户操作必须有授权和审计。
- 测试必须包含跨租户越权用例。

## 一致性策略

- 强一致：核心交易、余额、库存等不可破坏不变量。
- 最终一致：通知、统计、异步索引等可补偿场景。
- 幂等：重试、回调、批处理必须有幂等键或唯一约束。
- 乐观锁：适合并发更新同一业务对象。
