# 数据库设计综合方法

## 核心顺序

```text
业务事实
→ 实体和关系
→ 表结构和约束
→ 查询场景
→ 索引方案
→ 迁移和回滚
→ 生产操作安全
```

## 方法组合

| 目标 | 方法 |
|------|------|
| 表达业务事实 | 实体建模、关系基数、生命周期分析 |
| 保证数据质量 | 主键、外键、唯一约束、检查约束 |
| 支撑查询性能 | 访问模式、选择性、复合索引、执行计划 |
| 降低上线风险 | 兼容迁移、分批回填、灰度切换、回滚 |
| 降低生产风险 | 备份恢复、影响面评估、变更门禁 |

## 设计检查

- 数据库约束应兜住核心业务不变量。
- 索引设计必须来自查询场景和性能目标。
- 迁移计划必须包含存量数据、发布顺序和回滚。
- 多租户、权限、审计、脱敏要作为边界条件前置处理。
- 交付物必须能被后端、QA、DevOps/SRE 和安全工作流接手。
