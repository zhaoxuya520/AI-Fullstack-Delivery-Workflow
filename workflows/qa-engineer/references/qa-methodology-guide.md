# QA 测试方法论实战指南

> 面向 APP / 小程序 / 网页产品的功能测试、探索性测试、回归测试。

## 1. 测试策略选择

```text
┌─ 风险驱动测试策略 ─────────────────────────────────────────┐
│                                                             │
│  高风险 + 高频率 → 自动化 + 每次回归                       │
│  高风险 + 低频率 → 手工深度测试 + 关键路径自动化           │
│  低风险 + 高频率 → 自动化（低优先级）                      │
│  低风险 + 低频率 → 探索性测试（时间允许时）                │
│                                                             │
│  80/20 法则：20% 的功能承载 80% 的用户流量                 │
│  优先测试这 20%                                            │
└─────────────────────────────────────────────────────────────┘
```

## 2. 测试用例设计方法

| 方法 | 适用场景 | 示例 |
|------|----------|------|
| **等价类划分** | 输入有明确范围 | 年龄：<0/0-17/18-120/>120 |
| **边界值分析** | 有数值边界 | 最小值/最小值-1/最大值/最大值+1 |
| **判定表** | 多条件组合 | 优惠券：类型×金额×有效期×叠加规则 |
| **状态转换** | 有状态流转 | 订单：待支付→已支付→已发货→已收货→已退款 |
| **场景法** | 端到端业务流程 | 用户注册→浏览→加购→下单→支付 |
| **错误推测** | 经验驱动 | 空输入/超长/特殊字符/并发/弱网 |

## 4. 探索性测试框架

```text
Session-Based Exploratory Testing (SBET):

任务: 探索 [功能区域]，发现 [目标类型] 问题
时间: 30~60 分钟一个 session

探索维度（RCRCRC）:
  R - Recent（最近改动的代码）
  C - Core（核心功能）
  R - Risky（高风险区域）
  C - Configuration（配置相关）
  R - Repaired（修过的 Bug 区域）
  C - Chronic（老问题区域）

记录格式:
  - 发现时间
  - 操作步骤
  - 截图/录屏
  - 严重程度判断
  - 复现率
```

## 5. Bug 报告标准

```text
标题: [模块][严重程度] 简短描述

环境:
  - 设备: iPhone 15 / Chrome 120 / 微信 8.0
  - 系统: iOS 17.2 / Windows 11
  - 版本: v2.3.1 (build 456)
  - 网络: WiFi / 4G / 弱网

严重程度:
  P0-致命: 核心功能不可用/数据丢失/安全漏洞
  P1-严重: 主要功能异常/影响大量用户
  P2-一般: 次要功能异常/有绕过方案
  P3-轻微: UI瑕疵/文案错误/体验不佳

复现步骤:
  1. [精确步骤]
  2. [精确步骤]
  3. ...

预期结果: [应该怎样]
实际结果: [实际怎样]
附件: [截图/录屏/日志]
复现率: [必现/偶现(概率)]
```

## 6. 回归测试策略

```text
全量回归: 大版本发布前（2~3天）
冒烟回归: 每日构建（30分钟，10~15个核心case）
影响面回归: 修改后只跑相关模块（按代码变更范围）

自动化覆盖:
  核心业务流程（登录/注册/下单/支付）→ 100% 自动化
  CRUD 基础操作 → 自动化
  UI 细节/交互体验 → 手工探索

回归选择优先级:
  1. 修改的模块本身
  2. 调用修改模块的上游
  3. 被修改模块调用的下游
  4. 核心路径（即使没改动）
```

## 7. 移动端/小程序测试要点

```text
兼容性:
  iOS: 最新版 + 前两个大版本
  Android: 主流品牌 Top 5 + 系统 10~14
  微信小程序: 基础库版本覆盖 90% 用户
  浏览器: Chrome/Safari/Firefox/Edge + 微信内置

专项测试:
  □ 弱网（3G/断网恢复/切换网络）
  □ 中断（来电/锁屏/切后台/通知）
  □ 权限（相机/定位/相册拒绝后提示）
  □ 内存（长时间使用/大量数据）
  □ 横竖屏（如适用）
  □ 深色模式
  □ 字体大小（无障碍大字体）
  □ 安装/升级/卸载
  □ 推送通知
  □ 分享（微信/朋友圈/短信）
```

## 8. 验收标准检查清单

```text
□ 所有 P0/P1 用例通过
□ P2 用例通过率 ≥ 95%
□ 无未关闭的 P0/P1 Bug
□ 核心业务流程端到端通过
□ 兼容性测试通过（覆盖目标设备）
□ 性能满足 SLA（首屏/接口响应/帧率）
□ 安全扫描通过（无高危漏洞）
□ 回归测试通过（自动化 + 手工）
□ 上线 checklist 确认
```
