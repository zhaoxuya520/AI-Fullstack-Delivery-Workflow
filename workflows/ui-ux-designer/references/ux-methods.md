# UI/UX 方法参考

## 1. 用户任务分析

```text
用户是谁？
用户进入页面前发生了什么？
用户要完成什么任务？
完成任务需要哪些信息？
完成后用户要去哪里？
```

适合：所有页面设计的起点。

---

## 2. 用户流程

```text
入口
→ 查看信息
→ 执行操作
→ 系统反馈
→ 成功 / 失败
→ 后续动作
```

适合：多步骤任务、表单、审批、支付、管理后台。

---

## 3. 信息架构

```text
业务对象
→ 页面清单
→ 导航结构
→ 信息分组
→ 操作分组
→ 状态反馈
```

适合：多页面系统、后台、复杂业务模块。

---

## 4. 线框图

线框图关注结构而非视觉细节。

```text
页面区域
信息优先级
主操作位置
状态反馈位置
异常提示位置
```

适合：早期设计、快速评审、无设计工具时交接。

---

## 5. 组件化设计

```text
页面
→ 区块
→ 组件
→ 子组件
→ 状态
→ 事件
```

适合：前端交接、组件库复用、设计系统建设。

---

## 6. 状态设计

每个页面和组件都应考虑：

```text
默认
加载
空数据
错误
权限不足
提交中
成功
失败
禁用
```

适合：所有动态页面和表单。

---

## 7. 可用性评审

评审问题：

```text
用户知道自己在哪里吗？
用户知道下一步做什么吗？
用户能完成核心任务吗？
失败后用户能恢复吗？
页面是否只要求用户记忆，不给提示？
```

---

## 8. 设计交接

好的设计交接包含：

```text
页面清单
用户流程
组件清单
字段和数据需求
交互规则
状态矩阵
响应式规则
可访问性要求
验收标准
```
