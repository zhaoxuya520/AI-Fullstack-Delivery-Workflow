# 热门开源 UI 项目模式参考指南

## 1. 参考对象

```text
Ant Design：企业级中后台。
MUI：React + Material 组件生态。
Radix UI：无样式可访问交互原语。
shadcn/ui：Radix + Tailwind 的可复制组件模式。
Tailwind UI：应用 UI 布局块和页面模式。
Tabler / AdminLTE：开源管理后台模板。
```

---

## 2. 可借鉴的共同模式

### 组件优先

```text
页面不是一张图，而是组件组合。
每个组件有输入、状态、事件和验收标准。
```

### 可访问性内建

```text
Radix / shadcn 强调键盘、焦点和 ARIA。
设计说明应明确哪些组件需要焦点管理、快捷键、语义标签。
```

### 样式和行为解耦

```text
Radix 负责行为和可访问性。
Tailwind / shadcn 负责视觉和组合。
设计交接时要区分：交互行为、视觉样式、数据来源。
```

### 管理台信息密度

```text
Ant Design / Tabler / AdminLTE 强调表格、筛选、表单、侧边栏和数据操作。
设计时要定义表格列、筛选项、批量操作、分页和空状态。
```

---

## 3. 对本工作流的落地规则

```text
1. 所有页面先拆组件，再写视觉说明。
2. 所有交互组件必须说明键盘和焦点行为。
3. 表格、表单、导航、弹窗、下拉、Tab、Popover 必须有状态矩阵。
4. 如果项目使用 shadcn/Radix，应优先描述组件行为和组合方式，而不是只写像素级视觉。
5. 如果项目是后台系统，应优先参考 Ant Design / Carbon / Tabler 的表格和表单模式。
6. 如果项目是公开服务，应优先参考 USWDS / GOV.UK 的表单、校验和可访问性模式。
```
