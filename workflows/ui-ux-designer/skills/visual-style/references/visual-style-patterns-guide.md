# 视觉风格模式指南

## 目标

本指南用于帮助 UI/UX 设计工作流在页面设计前先确定视觉风格方向，避免只输出通用、模板化、AI 味很重的界面。

设计风格不是装饰层，而是：

```text
用户认知
→ 品牌气质
→ 信息密度
→ 交互反馈
→ 组件形态
→ 可访问性
→ 前端实现成本
```

---

## 使用原则

```text
1. 先匹配产品场景，再选择风格。
2. 先定义信息层级，再定义视觉效果。
3. 每种风格都必须落到颜色、排版、组件、状态和动效。
4. 不照抄流行视觉，只提炼可适配的规则。
5. 不为炫技牺牲可读性、性能和可访问性。
6. 不用默认 AI 生成感：大渐变、大圆角、漂浮卡片、无意义插画、假 3D、过度发光。
7. 反 AI 化不是去风格化，也不是把所有产品都做成纯白企业 SaaS。
8. 风格必须匹配品类、用户圈层和内容气质；动漫、游戏、潮流、创意产品可以更有表现力。
```

---

## 风格模式库

### 1. 液态玻璃 / Glassmorphism

适合：高端工具、AI 产品、创意工具、操作系统感页面、轻量 Dashboard。

关键特征：

```text
半透明层
背景模糊
细边框
高光描边
轻微层叠阴影
深浅背景切换
```

必须约束：

```text
文字必须放在足够稳定的承载层上。
背景不能过花。
透明层级最多 2-3 层。
重要按钮不要只靠透明和模糊区分。
移动端注意 backdrop-filter 性能。
```

避免 AI 味：

```text
不要所有卡片都玻璃化。
不要大面积彩色渐变背景加高斯模糊球。
不要用没有业务含义的漂浮圆球做装饰。
```

---

### 2. 极简 SaaS / Minimal SaaS

适合：B2B SaaS、管理后台、订阅型工具、数据产品。

关键特征：

```text
干净留白
强信息层级
克制色彩
清晰卡片
少量品牌色强调主操作
高可读性表单和表格
```

必须约束：

```text
主操作必须突出。
空状态必须有下一步行动。
表格必须处理筛选、排序、分页和批量操作。
颜色不超过主色、成功、警告、危险、中性色五组。
```

避免 AI 味：

```text
不要每个区块都是同款圆角卡片。
不要用过度统一的渐变图标。
不要让页面看起来像无行业属性的模板站。
```

---

### 3. 企业专业 / Enterprise Professional

适合：ERP、CRM、风控、权限系统、审批系统、内部管理平台。

关键特征：

```text
高信息密度
清晰分组
稳定导航
强状态标识
表格和筛选优先
操作可追溯
```

必须约束：

```text
信息密度高但不能拥挤。
危险操作必须有影响说明。
权限状态必须明确可见、禁用或隐藏。
状态颜色必须统一且有文本辅助。
```

避免 AI 味：

```text
不要把企业后台做成营销落地页。
不要用夸张插画替代真实数据状态。
不要为了“高级感”降低信息可读性。
```

---

### 4. 编辑排版 / Editorial

适合：官网、内容产品、品牌介绍、案例页、知识库、报告页。

关键特征：

```text
强标题层级
杂志式留白
大图或重点内容区
节奏化版式
高质量字体和段落排版
```

必须约束：

```text
排版层级要服务阅读路径。
标题、摘要、正文、引用、行动按钮必须区分。
移动端要重新组织节奏，不是简单缩小。
```

避免 AI 味：

```text
不要堆空泛大标题和抽象渐变背景。
不要用无意义的“未来感”文案填充视觉。
不要每屏都是相同 Hero + 卡片结构。
```

---

### 5. Bento Grid / 模块化卡片

适合：功能展示、产品首页、仪表盘、AI 工具能力介绍、控制台概览。

关键特征：

```text
不同尺寸的信息卡片
模块化布局
重点能力放大
图文混排
局部可视化
```

必须约束：

```text
卡片大小必须反映信息优先级。
每张卡片只表达一个主题。
移动端要定义堆叠顺序。
不能为了拼图牺牲阅读路径。
```

避免 AI 味：

```text
不要所有项目都套 Bento。
不要每张卡片都放发光图标和渐变背景。
不要让卡片内容像营销口号集合。
```

---

### 6. 新粗野 / Neo-brutalism

适合：开发者工具、创意社区、活动页、年轻化品牌、实验性产品。

关键特征：

```text
粗边框
高对比色
硬阴影
强几何
直接文字
故意不精致的张力
```

必须约束：

```text
只能作为明确品牌方向使用。
核心操作仍要清晰。
颜色冲突不能影响可读性。
复杂后台不建议大面积使用。
```

避免 AI 味：

```text
不要只复制粗黑边和彩色块。
不要让风格压过任务完成。
不要忽略状态和可访问性。
```

---

### 7. 触感拟物 / Tactile Skeuomorphism

适合：音乐、创作工具、金融卡片、设备控制、仪表盘、Apple 风格产品。

关键特征：

```text
材质感
细腻阴影
真实按钮反馈
微弱高光
可触摸的层次
```

必须约束：

```text
拟物只用于增强可理解性。
不要重回复杂真实材质堆叠。
交互反馈要轻，不要拖慢操作。
```

避免 AI 味：

```text
不要使用廉价 3D 质感图标。
不要用随机金属、玻璃、塑料材质混搭。
不要让视觉资产看起来像生成图素材包。
```

---

### 8. 暗黑科技 / Dark Tech

适合：安全平台、开发者工具、监控系统、AI 控制台、数据分析平台。

关键特征：

```text
深色背景
高对比文本
冷色强调
代码/数据感
细线分割
状态色明确
```

必须约束：

```text
正文对比度必须足够。
红色不要滥用，避免误导为告警。
图表颜色要可区分。
长时间使用要避免视觉疲劳。
```

避免 AI 味：

```text
不要默认黑底 + 蓝紫渐变 + 发光网格。
不要滥用霓虹、粒子、HUD 装饰。
不要把安全产品做成游戏 UI。
```

---

### 9. 数据密集 / Data-dense Dashboard

适合：运营分析、BI、监控、交易、日志、安全态势、财务系统。

关键特征：

```text
信息密度高
图表和表格并重
筛选器清晰
状态颜色稳定
时间范围和刷新状态明确
```

必须约束：

```text
先定义数据问题，再选择图表。
图表必须有空值、错误、加载和权限状态。
颜色不能只靠色相表达含义。
移动端要降级为重点指标和详情页。
```

避免 AI 味：

```text
不要放没有真实含义的装饰性图表。
不要使用随机曲线和假数据做视觉重点。
不要所有指标都用大数字卡片。
```

---

### 10. 开发者工具 / Developer Tool

适合：API 平台、SDK 文档、DevOps、CLI、日志、云服务、配置系统。

关键特征：

```text
代码优先
清晰侧边导航
命令示例
状态反馈直接
搜索和复制操作突出
```

必须约束：

```text
示例必须真实可信。
错误信息要可复制和可定位。
空状态要给下一步命令或配置入口。
文档、控制台和 API 字段命名要一致。
```

避免 AI 味：

```text
不要用抽象 AI 插画替代真实命令和示例。
不要让开发者页面看起来像普通营销页。
不要牺牲密度和可复制性。
```

---

### 11. 复古 / Retro / Y2K

适合：活动页、品牌营销、游戏、创意社区、个性化产品。

关键特征：

```text
怀旧字体
像素/窗口元素
高饱和色
复古控件
有意的时代感
```

必须约束：

```text
只适合明确品牌场景。
核心路径仍要现代化、可读、可访问。
表单和支付等关键流程不建议过度复古。
```

避免 AI 味：

```text
不要混搭所有复古元素。
不要使用随机贴纸和无意义装饰。
不要让风格成为噪音。
```

---

### 12. 温暖人文 / Human Warmth

适合：教育、健康、家庭、社区、公益、个人效率工具。

关键特征：

```text
柔和色彩
自然插画或真实照片
圆润但不过度的组件
温和反馈
可理解文案
```

必须约束：

```text
情绪表达要符合场景。
不要为了温暖牺牲任务效率。
错误和风险提示仍要明确。
```

避免 AI 味：

```text
不要使用千篇一律的扁平人物插画。
不要用空泛鼓励文案替代具体帮助。
不要所有页面都用大面积柔和渐变。
```

---

### 13. 高级奢华 / Premium Luxury

适合：高端品牌、金融财富、精品服务、作品集、艺术和设计产品。

关键特征：

```text
克制配色
大留白
精致字体
低饱和或黑白金属色
慢节奏动效
高质量图像
```

必须约束：

```text
少即是多。
图片和文案质量必须匹配风格。
交互不能过慢。
可读性优先于氛围。
```

避免 AI 味：

```text
不要用假高级的黑金渐变。
不要堆廉价金属材质和光效。
不要让页面像模板化奢侈品落地页。
```

---

### 14. 动漫 / IP / ACG 风格

适合：动漫社区、IP 官网、二次元工具、游戏活动、角色内容、粉丝运营页面。

关键特征：

```text
角色视觉
鲜明色彩
情绪化排版
贴纸/徽章/卡片化内容
剧情感或世界观氛围
更强的视觉记忆点
```

必须约束：

```text
视觉表现要服务 IP 气质和用户圈层。
角色、配色、符号必须有来源，不能随机拼贴。
核心操作仍要清楚，不能被装饰遮挡。
表单、支付、账号、安全等关键流程要降低装饰强度。
移动端要控制图片体积和动效性能。
```

避免同质化：

```text
不要套纯白企业 SaaS 模板。
不要把所有动漫产品都做成粉紫渐变。
不要使用无授权或无来源的角色感素材。
不要为了“反 AI 味”去掉品类该有的情绪和表现力。
```

---

### 15. 游戏 / 沉浸式活动风格

适合：游戏官网、活动页、战绩系统、赛事页面、社区任务、成就系统。

关键特征：

```text
强氛围背景
任务/成就/等级感
徽章和状态反馈
更强动效
沉浸式视觉层级
```

必须约束：

```text
沉浸感不能牺牲操作效率。
常用功能入口必须稳定可见。
奖励、等级、状态要有明确规则说明。
登录、支付、隐私、安全提示必须清楚。
```

避免同质化：

```text
不要把所有游戏页面都做成暗黑霓虹 HUD。
不要用无意义粒子和光效替代玩法信息。
不要让界面像概念海报而不是可用产品。
```

---

### 16. 政务/可信赖 / Civic Trust

适合：政务、公共服务、医疗、教育机构、合规系统。

关键特征：

```text
清晰结构
高可读性
稳定配色
明确表单
强辅助说明
可访问性优先
```

必须约束：

```text
可信比炫酷重要。
所有提示必须明确、可操作。
错误恢复路径必须清楚。
不要隐藏关键条款和限制。
```

避免 AI 味：

```text
不要用科技感包装严肃服务。
不要过度品牌化导致不可信。
不要用抽象插画替代明确说明。
```

## 风格选择矩阵

| 场景 | 优先风格 | 谨慎使用 |
|---|---|---|
| 企业后台 | Enterprise Professional / Minimal SaaS | Neo-brutalism / 过强 Glassmorphism |
| AI 工具 | Minimal SaaS / Glassmorphism / Bento | 黑紫发光“AI 模板感” |
| 安全平台 | Dark Tech / Data-dense Dashboard / Developer Tool | 游戏 HUD / 霓虹粒子 |
| 开发者平台 | Developer Tool / Minimal SaaS | 过度营销式 Hero |
| 内容官网 | Editorial / Premium / Human Warmth | 卡片堆砌模板 |
| 数据分析 | Data-dense Dashboard / Enterprise Professional | 装饰性假图表 |
| 移动 App | Human Warmth / Tactile / Minimal | 复杂玻璃层叠 |
| 动漫/IP/ACG | ACG 风格 / Human Warmth / Bento / Retro | 纯白企业 SaaS / 无来源角色拼贴 |
| 游戏/活动 | 游戏沉浸式 / Dark Tech / Neo-brutalism / Retro | 通用暗黑霓虹 HUD / 概念海报化 |
| 政务医疗 | Civic Trust / Enterprise Professional | 赛博科技感 |

```text

色彩规则：
字体和排版：
圆角、阴影、边框：
图标和插画：
组件风格：
状态风格：
动效规则：
响应式影响：
可访问性风险：
前端实现注意：
反 AI 化检查：
```
