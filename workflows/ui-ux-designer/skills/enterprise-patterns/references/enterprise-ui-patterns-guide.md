# 企业级 UI 模式参考指南

## 1. 适用范围

适合参考 Ant Design、Carbon、Fluent、Atlassian、Polaris、Primer、Pajamas 等设计系统中的企业级产品模式。

典型产品：

```text
管理后台
SaaS 控制台
数据看板
开发者平台
电商商家后台
DevOps 平台
权限和审批系统
```

---

## 2. 企业级页面的核心特征

```text
信息密度高
角色权限复杂
表单和表格多
批量操作多
错误和异常场景多
需要审计、确认和回滚
需要前端、API、QA、安全强交接
```

---

## 3. 页面类型模式

| 页面类型 | 设计重点 | 必须补充状态 |
|---|---|---|
| 列表页 | 搜索、筛选、排序、分页、批量操作 | 加载、空、无结果、错误、权限不足 |
| 详情页 | 信息分组、主次关系、操作入口 | 加载、缺失字段、无权限、历史记录 |
| 表单页 | 字段分组、校验、保存、退出保护 | 输入错误、提交中、提交失败、草稿 |
| Dashboard | 指标层级、时间范围、异常提示 | 数据加载、无数据、数据延迟、权限不足 |
| 设置页 | 分组、危险操作、说明文本 | 禁用、保存成功/失败、回滚 |
| 审批页 | 决策依据、操作确认、状态流转 | 待处理、已通过、已拒绝、撤回、超时 |

---

## 4. 管理后台布局

```text
顶部：全局搜索、账号、通知、环境标识
侧边栏：一级/二级导航，当前模块高亮
内容头部：页面标题、说明、主操作
筛选区：搜索、筛选、视图切换
主内容：表格、卡片、表单、图表
反馈区：toast、banner、inline error
底部：分页、批量操作、保存栏
```

---

## 5. 设计检查

```text
是否支持不同权限角色？
是否定义了批量操作和危险操作？
表格是否有空状态和无结果状态？
表单是否有字段级错误？
页面是否有全局反馈和局部反馈？
移动端或窄屏是否有退化方案？
前端是否能直接识别组件和状态？
API 是否能识别字段和错误场景？
```
