# 表单、表格和空状态专项指南

## 1. 表单设计

参考 Atlassian、USWDS、Carbon、Polaris、Ant Design 的共同模式。

### 必须定义

```text
字段标签
帮助文本
必填 / 选填
默认值
输入约束
字段级校验
表单级错误
提交中状态
提交成功状态
提交失败状态
离开页面保护
```

### 错误提示规则

```text
告诉用户哪里错。
告诉用户为什么错。
告诉用户怎么改。
保留用户已输入内容。
错误文本不能只靠颜色表达。
```

## 2. 数据表格设计

参考 Ant Design、Carbon、MUI Data Grid、GitHub Primer 和管理后台项目。

### 必须定义

```text
列名和字段来源
排序规则
筛选规则
搜索规则
分页或无限滚动
行操作
批量操作
选择状态
加载状态
空数据状态
无搜索结果状态
错误状态
权限限制
移动端退化方案
```

### 表格状态区分

```text
空数据：系统还没有任何数据，应引导创建。
无结果：筛选/搜索后没有匹配，应引导清除条件。
加载失败：数据请求失败，应允许重试。
无权限：用户不能查看，应解释原因或申请路径。
```

## 3. 空状态设计

参考 Material、Atlassian、Polaris、Carbon、GitLab Pajamas。

### 空状态结构

```text
标题：说明当前状态。
说明：解释为什么为空。
行动：给出下一步。
辅助：必要时给文档、导入、示例或联系入口。
```

### 空状态类型

| 类型 | 说明 | 推荐动作 |
|---|---|---|
| 首次使用为空 | 用户还没创建内容 | 创建 / 导入 / 查看示例 |
| 搜索无结果 | 当前条件没有匹配 | 清除筛选 / 修改关键词 |
| 权限为空 | 用户无权查看 | 申请权限 / 联系管理员 |
| 错误导致为空 | 系统加载失败 | 重试 / 查看状态 / 联系支持 |
| 功能未启用 | 需要配置或开关 | 去配置 / 了解更多 |

## 4. 交接给前端/API/QA

```text
给前端：字段、状态、操作、响应式、可访问性。
给 API：字段来源、筛选排序、分页、错误码、权限场景。
给 QA：表单校验、表格状态、空状态类型、错误恢复路径。
```
