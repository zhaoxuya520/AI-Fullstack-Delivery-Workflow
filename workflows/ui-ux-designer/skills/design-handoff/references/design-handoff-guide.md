# 设计交接指南

## 1. 交接目标

设计交接不是只给截图，而是让前端、API、QA 和安全工作流能直接理解页面结构、交互状态、字段来源和验收标准。

---

## 2. 交给前端

```text
页面清单
页面结构
组件清单
组件状态
交互规则
响应式要求
可访问性要求
验收标准
```

前端拿到后应能拆出组件和页面任务。

---

## 3. 交给 API 设计

```text
页面需要展示哪些字段
用户会触发哪些动作
每个动作需要什么输入
需要哪些错误场景
权限如何影响展示和操作
```

API 设计拿到后应能输出资源、接口、字段和错误码。

---

## 4. 交给 QA

```text
主路径
失败路径
状态矩阵
权限场景
响应式场景
可访问性检查点
```

QA 拿到后应能设计测试用例。

---

## 5. 交给安全

```text
敏感信息是否展示
危险操作是否二次确认
权限不足如何展示
是否有越权入口
是否有隐私提示或审计要求
```

安全工作流拿到后应能识别 UI 层风险。

---

## 6. 交接验收标准

```text
页面能否被前端拆成任务？
字段能否被 API 设计识别？
状态能否被 QA 测试？
权限和敏感数据能否被安全评审？
待确认问题是否列出？
```
