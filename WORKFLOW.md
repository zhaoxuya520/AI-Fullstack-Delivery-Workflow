# 全栈交付生命周期工作流

## 1. 生命周期总览

```text
Idea / Business Goal
    ↓
产品经理工作流
    ↓
项目经理工作流
    ↓
UI/UX 设计工作流
    ↓
API 设计工作流 + 数据库工程师工作流
    ↓
前端工程师工作流 + 后端工程师工作流
    ↓
测试工程师工作流 + 自动化测试工作流
    ↓
安全工程师工作流 + 逆向/渗透工作流
    ↓
DevOps 工程师工作流
    ↓
SRE/运维工作流
    ↓
数据分析工作流 / AI 算法工作流
    ↓
技术文档工作流
    ↓
Field Journal
```

## 2. 阶段产物

| 阶段 | 主工作流 | 核心产物 |
|---|---|---|
| 想法澄清 | 产品经理 | PRD、MVP、用户故事、验收标准 |
| 计划拆解 | 项目经理 | 任务列表、里程碑、风险清单 |
| 体验设计 | UI/UX | 用户流程、页面结构、组件清单 |
| 接口与数据 | API、数据库 | API 契约、ER/DDL、迁移方案 |
| 功能实现 | 前端、后端、全栈 | 页面、接口、业务逻辑、联调结果 |
| 质量验证 | QA、自动化测试 | 测试用例、测试报告、自动化脚本 |
| 安全验证 | 安全、逆向/渗透 | 安全报告、漏洞复现、修复建议 |
| 部署上线 | DevOps | CI/CD、部署文档、回滚方案 |
| 稳定性 | SRE | 监控、日志、告警、故障复盘 |
| 数据智能 | 数据分析、AI/算法 | 指标、报表、模型/RAG 方案 |
| 文档沉淀 | 技术文档 | README、API 文档、架构文档、复盘 |

## 3. 单工作流与多工作流判断

### 单工作流处理

适用于边界清晰、产物单一的任务：

```text
修复一个 CSS 问题 → 前端工程师工作流
补一个接口测试 → 自动化测试工作流
写部署说明 → 技术文档工作流
优化一个 SQL → 数据库工程师工作流
```

### 多工作流协同

适用于跨层交付或影响多个系统的任务：

```text
新增用户管理模块
→ 产品经理
→ API 设计
→ 数据库
→ 后端
→ 前端
→ QA
→ 技术文档
```

```text
上线前安全检查
→ 安全工程师
→ 逆向/渗透
→ 后端/前端修复
→ QA 回归
→ DevOps 部署
→ 技术文档
```

## 4. 执行闭环

每个任务必须形成闭环：

```text
目标明确
→ 输入齐全
→ 路由正确
→ 过程可追踪
→ 结果可验证
→ 文档可复用
→ 经验可沉淀
```
